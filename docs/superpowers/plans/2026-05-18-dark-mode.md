# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an auto-activating dark mode to the personal site via `prefers-color-scheme`, with light mode visually unchanged.

**Architecture:** Single-file CSS change. Hardcoded colors are lifted into seven `:root` custom properties at their current (light) values, every color declaration is rewired to `var(--…)`, and one `@media (prefers-color-scheme: dark)` block redefines the seven tokens with dark values. No HTML/JS changes, no toggle, no persisted state.

**Tech Stack:** Plain CSS (`styles/main.css`). Static site, no build step, no test runner.

Spec: `docs/superpowers/specs/2026-05-18-dark-mode-design.md`

- [ ] **Step 2: Add the `:root` token block at the very top of `styles/main.css`**

Insert this block as the new first lines of the file, before the existing `*, *::before, *::after` rule:

```css
:root {
  --bg: #ffffff;
  --accent: #1d4ed8;
  --accent-link: #3b82f6;
  --muted: #6b7280;
  --faint: #9ca3af;
  --border: #e5e7eb;
  --border-input: #d1d5db;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0a0a0a;
    --accent: #5b8def;
    --accent-link: #6b9aef;
    --muted: #9aa3b2;
    --faint: #6b7280;
    --border: #2a2a2a;
    --border-input: #3a3a3a;
  }
}

```
